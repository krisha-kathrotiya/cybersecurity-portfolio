# Final Security Audit Report – Botium Toys

## Objective
The purpose of this audit was to review the current security controls at Botium Toys and identify any missing or weak areas based on the provided risk assessment.

## Key Findings

The audit identified several important issues in the current security setup.

- Sensitive customer data, including credit card information, is not encrypted.
- There are no backups or disaster recovery plans in place.
- All employees have access to internal data, and least privilege is not enforced.
- There is no intrusion detection system (IDS) to monitor suspicious activity.

Some areas are partially implemented but still need improvement.

- A password policy exists, but it is weak and does not follow modern standards.
- There is no centralized password management system.
- Legacy systems are monitored manually without a proper schedule.

The organization does have some controls in place.

- A firewall is implemented to filter network traffic.
- Antivirus software is installed and monitored.
- Physical security controls such as locks, CCTV, and fire systems are in place.

## Risk Level
Overall, the risk level is high due to missing controls in access management, data protection, monitoring, and recovery.

## Conclusion
Botium Toys has basic security measures, but several critical controls are missing. These gaps increase the risk of data breaches and operational issues. Improvements are needed in access control, encryption, monitoring, and recovery planning to strengthen the organization’s security.
