# Security Audit – Botium Toys (Simulated)

## Overview
This project documents a simulated internal security audit conducted for a fictional company, Botium Toys.  
The audit was completed as part of a cybersecurity training program and demonstrates my ability to assess security controls, identify risks, and document findings using structured audit methods.

All organizations, systems, and scenarios presented here are fictional and created for educational and portfolio purposes only.

## Audit Scope and Objectives
The audit focused on:
- Identifying assets and potential security risks
- Reviewing existing security controls
- Evaluating risks related to confidentiality, integrity, and availability (CIA triad)
- Documenting findings using a structured risk assessment checklist

## Audit Components
This audit includes the following components:
1. Scope, goals, and risk assessment overview
2. Control categories and control evaluation
3. Risk assessment checklist with documented findings and recommendations

## Evidence Reviewed
The audit was supported by simulated documentation and asset records provided as part of the scenario, including:
- Asset inventory
- Risk assessment documentation
- Control category references

## Repository Structure
- `assets/` – Audit-related documents and supporting materials
- `evidence-notes.md` – Notes linking evidence to audit findings
- `README.md` – Project overview and methodology

- Risk assessment and documentation
- Security control analysis
- Audit reporting and evidence handling
